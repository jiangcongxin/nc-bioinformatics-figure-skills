## Use Rules

- Use this inventory for routing and function design, not as proof that every package target already exists.
- Use `implemented in ncfigR` only for functions documented in `ncfigR_api.md`.
- Use `stub needed` for clear domain functions that should be created in future R packages.
- Use `package planned` for package-level routing before implementation.
- Keep `license_note` visible when generating project-transfer code.
